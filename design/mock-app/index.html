Start with no fleet, no repo, and no deployment definitions.

The first-run UI gives the user a safe sequence: create storage, define hierarchy levels and first instances, set the enterprise target standard, assign a component to a device, and render a lab bundle before any live apply is possible.

3. Set the enterprise target standard

Most organizations standardize on one target family for deployments. Per-deployment platform exceptions are supported, but they should be governed and rare.

4. Add components to the selected edge

Choose the hierarchy instance where components will run, then check the component templates to assign to that edge. Each checked row creates or updates a component instance for the selected deployment scope.
